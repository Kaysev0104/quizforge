QuizForge Live — Setup Guide

Get your live quiz platform running in ~20 minutes with a free Supabase backend.

📁 Project Files

quizforge/
├── index.html     ← Landing page (host login + participant join)
├── host.html      ← Host dashboard (upload, manage room, live view)
├── participant.html ← Participant view (answer questions)
├── config.js      ← ⭐ YOUR SUPABASE KEYS GO HERE
├── supabase.js    ← Supabase client bootstrap
└── setup.html     ← This guide

1 Create a Free Supabase Project

  1. Go to supabase.com and sign up (free)
  2. Click New Project, give it a name like quizforge
  3. Choose a region close to your users, set a strong database password
  4. Wait ~2 minutes for the project to spin up

2 Get Your API Keys

  1. In your Supabase project, go to Settings → API
  2. Copy the Project URL (looks like https://xyzabc.supabase.co)
  3. Copy the anon / public key (long string starting with eyJ…)
  4. Open config.js and paste them in:
const SUPABASE_URL  = 'https://YOUR_PROJECT.supabase.co';
const SUPABASE_ANON = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';

3 Create the Database Tables

Go to Supabase → SQL Editor → New Query and run this SQL:

-- ── ROOMS ──────────────────────────────────────────────────
CREATE TABLE rooms (
  id                  UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  pin                 TEXT NOT NULL UNIQUE,
  host_id             UUID REFERENCES auth.users(id),
  quiz_title          TEXT DEFAULT 'My Quiz',
  status              TEXT DEFAULT 'waiting',   -- waiting | active | ended
  questions           TEXT,                      -- JSON array
  current_question_index INT DEFAULT -1,
  timer_seconds       INT DEFAULT 30,
  show_leaderboard    BOOLEAN DEFAULT true,
  allow_late_join     BOOLEAN DEFAULT true,
  created_at          TIMESTAMPTZ DEFAULT NOW()
);

-- ── PARTICIPANTS ────────────────────────────────────────────
CREATE TABLE participants (
  id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  room_id         UUID REFERENCES rooms(id) ON DELETE CASCADE,
  nickname        TEXT NOT NULL,
  score           INT DEFAULT 0,
  correct_count   INT DEFAULT 0,
  partial_count   INT DEFAULT 0,
  joined_at       TIMESTAMPTZ DEFAULT NOW()
);

-- ── RESPONSES ───────────────────────────────────────────────
CREATE TABLE responses (
  id               UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  room_id          UUID REFERENCES rooms(id) ON DELETE CASCADE,
  participant_id   UUID REFERENCES participants(id) ON DELETE CASCADE,
  question_index   INT NOT NULL,
  answer           TEXT,
  is_correct       BOOLEAN DEFAULT false,
  is_partial       BOOLEAN DEFAULT false,
  points_awarded   INT DEFAULT 0,
  score_after      INT DEFAULT 0,
  answered_at      TIMESTAMPTZ DEFAULT NOW()
);

-- ── INDEXES ─────────────────────────────────────────────────
CREATE INDEX idx_participants_room ON participants(room_id);
CREATE INDEX idx_responses_room    ON responses(room_id);
CREATE INDEX idx_responses_part    ON responses(participant_id);
CREATE INDEX idx_rooms_pin         ON rooms(pin);

4 Set Row Level Security Policies

Still in SQL Editor, run this to allow the right access:

-- Enable RLS
ALTER TABLE rooms        ENABLE ROW LEVEL SECURITY;
ALTER TABLE participants ENABLE ROW LEVEL SECURITY;
ALTER TABLE responses    ENABLE ROW LEVEL SECURITY;

-- Rooms: hosts can manage their own; everyone can read active rooms
CREATE POLICY "Hosts manage own rooms"    ON rooms FOR ALL    USING (auth.uid() = host_id);
CREATE POLICY "Anyone can read rooms"     ON rooms FOR SELECT USING (true);

-- Participants: anyone can insert/read
CREATE POLICY "Anyone can join"           ON participants FOR INSERT WITH CHECK (true);
CREATE POLICY "Anyone can read parts"     ON participants FOR SELECT USING (true);
CREATE POLICY "Parts update own score"    ON participants FOR UPDATE USING (true);

-- Responses: anyone can insert/read
CREATE POLICY "Anyone can respond"        ON responses FOR INSERT WITH CHECK (true);
CREATE POLICY "Hosts can read responses"  ON responses FOR SELECT USING (true);

5 Enable Realtime

  1. Go to Supabase → Database → Replication
  2. Make sure Realtime is enabled for: rooms, participants, responses
  3. Also go to Authentication → Providers and make sure Email is enabled
💡 Realtime is what makes participant lists and response counts update live on the host screen without refreshing.

6 Deploy to Vercel (Free)

  1. Push your quizforge/ folder to a GitHub repository
  2. Go to vercel.com, sign in with GitHub
  3. Click Add New Project → import your repo
  4. Leave all settings default (it's a static site) → click Deploy
  5. Vercel gives you a URL like quizforge.vercel.app
✅ That's it! Your quiz platform is live and accessible from any device worldwide.

🔄 Alternative: Run Locally

If you just want to test locally, you need a simple HTTP server (opening HTML files directly won't work with Supabase auth):

# Option A — Python (installed on most Macs/Linux)
cd quizforge
python3 -m http.server 3000
# Open http://localhost:3000

# Option B — Node.js (npx, no install needed)
npx serve quizforge -p 3000

# Option C — VS Code Live Server extension
# Right-click index.html → "Open with Live Server"

⚡ How Real-time Works

QuizForge uses two Supabase real-time mechanisms together:

  • Broadcast — Host sends events (new question, reveal, quiz end) directly to all participants via WebSocket channel. Zero DB writes needed, nearly instant.
  • Postgres Changes — When a participant joins or submits a response, the DB change triggers a real-time event on the host's dashboard automatically.

🗺 Full Session Flow

HOST                                    PARTICIPANTS
────                                    ────────────
1. Upload CSV/Excel quiz
2. Create Room → gets PIN ─────────────→ Enter PIN + Nickname
3. See participants join live ←──────── Inserted into DB → host notified
4. Click "Start Quiz"
5. Question broadcast ─────────────────→ All participants see Q simultaneously
6. Response bar fills live ←──────────── Answers saved to responses table
7. Click "Reveal Answer" ──────────────→ Correct answers highlighted
8. Click "Next Question" ──────────────→ Next question loads (with timer)
   [repeat steps 5-8 for each Q]
9. Click "End Quiz" ───────────────────→ Final scores shown
10. Export Results (.xlsx)              Leaderboard + personal score
